gitban is git-native kanban for AI coding agents. Cards are markdown, state is the filename, the board is your repo. A validation gate blocks any card marked "done" that isn't — so a pipeline of disposable agents ships real work instead of confident fiction.

Stigmergic, not orchestrated

There's no scheduler and no central controller. Agents read the board, take a card, and write back what they did. Coordination emerges from the trail — like ants laying pheromones, not managers handing out tickets.

The trail

Disposable agents

Each agent picks up one card, does the work, and exits. No long-running agent straining to hold everything in context. The next one reads what the last one left behind.

The gate

Validation at the boundary

Cards live in templates with required headings and checkbox labels. The MCP server validates on create and on complete. "Just trust me, it's done" doesn't pass.

The substrate

The filesystem is the bus

State is in the filename. No database, no sync, no migrations. Disaster recovery is git reset. The whole history of how it was built is browsable, not lost to a ticketing tool.

FAQ

Is it really free?

Yes — free for any individual's own work, and free for non-commercial and open-source projects regardless of how many people are involved. Public repositories carrying a .gitban/ folder stay free forever. If you have a use that should be free and isn't, ask us.

When do I need to pay?

Only when gitban is used by or within an organization for commercial work. That's the single line — a commercial-use license, not a feature wall. A 500-person open-source project is free; a one-person company doing commercial work needs a paid license. The free and paid builds are the same software; you're buying the right to use it commercially, plus support.

Is it open source?

No. gitban ships as a closed binary — no public source drop, no OSI license. For Enterprise buyers who must audit during procurement, source is disclosed under NDA for security audit only (read, build, verify — never use, modify, or redistribute). It runs entirely offline with zero telemetry, so there's no phone-home behavior to inspect.

Does it phone home?

No. gitban runs entirely offline. License checks are a local, offline signature check against a key on your machine — no network call. Enterprise buyers can verify this directly during an NDA'd source audit.

How is a license tied to me?

A license is per seat = per machine. The key sits in your user config directory and is never committed to a repo. The free Solo build needs no key at all — it boots free when none is present.

Can I contribute to a project that uses gitban?

Always — and you never need gitban installed. The .gitban/ state in any repo is plain, human-readable markdown you can read and edit with any editor. The folder signals capability; it never gates contribution.
